Four gates decide whether it ships

This is the difference between a script generator and an engineer: a passing status code is not success.

Will it even send?Proves every variable resolves before spending a run.
Is anyone logged in?Catches the 200 that quietly returns the login page.
Did the work happen?Checks the business outcome, not just the response code.
Are these the right bytes?Verifies the exact file handed to you, and fingerprints it.
It changes its mind

When fixes run out it re-plans and rebuilds a different way, rather than patching harder.

It remembers

Only fixes proven green are kept — and applied before the first request next time.

It costs nothing to run

The whole pipeline is deterministic and local. AI is optional and off by default.
