Recordings go in at the top. Every stage below adds evidence, and four gates refuse to let an unproven script through. Nothing here needs an LLM — the gold, dashed modules are the only ones that ever call one, and they stay off unless you turn them on.

The auth wall — the 200 that lies semantic-triage.js → final-green-gate.js

An unauthenticated app doesn't answer 401; it answers 200 with the login page, and JMeter passes every sampler. If a response looks like a login page where the recording's did not, the session is gone and no number from that run means anything. This one fails the gate outright.

Green gate — is it really green? final-green-gate.js

Green means the shipped script passed, verified the business outcome, and matched the recording — judged only on the samplers that actually ship. A passing HTTP status is not enough.

Ship-point sanitizer — the bytes you open final-artifact.js

Checks the exact file being handed to you: reverts any corrupted substitution, re-asserts every fold, then fingerprints it — so if anything rewrites your script later (a JMeter save from a stale tab), the next run says so by name.

The loop back returns to Build

When repairs are exhausted the agent changes approach rather than patching harder: it re-plans, regenerates with a different strategy, and verifies the new script like any other attempt — bounded, and every attempt recorded. The dashed modules are the only ones that ever call an LLM, and only with AI assist switched on.

Memory spans every run

Only fixes that were verified green are remembered. On the next visit to the same app they are applied before the first request, so the agent doesn't re-suffer a problem it already solved — and any lesson whose request still fails loses confidence.
